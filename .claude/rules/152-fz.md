# Правило: соответствие 152-ФЗ

Эти требования — жёсткие. Любой код, который их нарушает, считается багом и блокируется
security-agent. Я не заменяю юриста: формулировки согласий, политику и оценку уровня
защищённости (УЗ) подтверждает специалист по ПДн. Но архитектура держится этих правил.

## 1. Локализация (ст. 18 ч. 5)
- Postgres, Storage и деплой — только в РФ (self-hosted Supabase: Yandex Cloud / VK Cloud /
  Selectel; S3 — Yandex Object Storage / VK Cloud S3).
- ЗАПРЕЩЕНО в коде/конфигах: Supabase Cloud (`*.supabase.co`), Vercel, Firebase, Stripe,
  Google Analytics/Tag Manager, зарубежные CDN для фото.
- Connection-строки и хосты деплоя должны указывать на РФ-инфраструктуру.

## 2. Без биометрии (ст. 11)
- Никакой детекции/распознавания лиц, тегирования по лицам, эмбеддингов лиц, vision-API.
- Появится распознавание → это отдельный строгий режим (письменное согласие, возможно ЕБС/
  572-ФЗ). До тех пор — запрет на уровне кода.

## 3. Согласие и журнал
- Согласие гостя фиксируется в `consents` (subject_uid, type, purpose, policy_version,
  granted_at, ip, user_agent) ДО первого кадра. Чекбокс не предзаполнен.
- Отзыв согласия → каскадное удаление фото субъекта (строки + объекты Storage),
  `consents.revoked_at`.

## 4. Хранение и доступ
- Фото отдаются ТОЛЬКО подписанным URL с TTL. Бакет `event-photos` приватный.
- Минимизация: хранится кадр с применённым фильтром, без оригинала. Лишних PII-полей нет.
- Шифрование at-rest (Object Storage) и in-transit (TLS) обязательно.

## 5. Сроки хранения (ст. 5)
- У каждого события `expires_at` (из `plans.retention_days`). По истечении — пометка
  `deleted` + физическая чистка Storage (`purge-expired`). Не хранить дольше необходимого.

## 6. Трансграничная передача (ст. 12)
- Допустима только APNs (пуши iOS); отражена в уведомлении РКН; состав данных минимален.
- Аналитика — только российская (AppMetrica/Метрика).

## 7. Права субъектов
- Поддержать доступ/уточнение/удаление/отзыв через `deletion_requests` и privacy-эндпойнты.

## Перед началом обработки данных (организационное, вне кода)
- Уведомление РКН подано; политика обработки ПДн опубликована; формы согласий внедрены;
  уровень защищённости определён, меры Приказа ФСТЭК №21 внедрены.
