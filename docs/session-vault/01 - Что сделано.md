# 01 — Что сделано

[[00 - Карта проекта|← к карте]]

## Фаза 1 — Backend (была до сессии + доработки)

- 9 таблиц Postgres, RLS на каждой ([[03 - Бэкенд (Supabase)]]).
- Security-definer функции: `is_event_host`, `current_guest_id`, `is_event_revealed`.
- Приватный бакет `event-photos`.
- **Доработка в сессии:** настроен Auth в `config.toml` — анонимные сессии гостей + email-OTP хоста (локально через Inbucket). См. [[05 - Инварианты 152-ФЗ]].

## Фаза 2 — Гостевой вход

- Edge Function `join-event` — вступление по `short_code`, запись согласия (IP/UA) **до** создания гостя, идемпотентность по `(event_id, auth_uid)`, лимит гостей из `plans`.
- Миграция `0003_join_guest_rpc.sql` — атомарное вступление под advisory-lock (защита от гонки на лимите гостей).
- Web-экраны: лендинг события, форма имя+согласие (чекбокс **не предзаполнен**), заглушка камеры.
- ✅ Проверено в браузере (Playwright): вступление 201, повтор 200, без согласия 422, согласие реально блокирует кнопку.

## Фаза 3 — Камера + загрузка + галерея

**Бэкенд (4 Edge Functions):**
- `upload-url` — подписанный PUT-URL, лимит кадров из `events.shots_per_guest`.
- `confirm-upload` — подтверждение загрузки.
- `photo-url` — подписанный GET-URL с проверкой видимости («проявка»).
- `public-event` — превью события по коду без PII.

**Фронтенд:**
- `lib/filters.ts` — 4 плёночных фильтра на canvas (film35, vintage, bw, summer), ресайз до 2048px, лимит 12МБ.
- `lib/photos.ts` — клиент загрузки/галереи, кэш подписанных URL.
- Экран камеры (`getUserMedia`), галерея «Мои/Общая».

✅ **Проверено end-to-end в браузере и на реальном телефоне (через HTTPS):**
- снять кадр → фильтр → upload-url → загрузка → confirm → галерея;
- «проявка»: до `reveal_at` гость видит только своё фото, после — все.
