# 02 — Архитектура и стек

[[00 - Карта проекта|← к карте]]

## Стек

| Слой | Технология |
|---|---|
| iOS (хост) | Swift / SwiftUI + Core Image (фильтры) — *ещё не начато* |
| Web/PWA (гость) | Next.js 15.5 App Router + TypeScript, `getUserMedia` |
| Backend | self-hosted Supabase (Postgres, Auth, Storage, Edge Functions/Deno) |
| Хостинг/БД/S3 | Yandex Cloud / VK Cloud (РФ) — *на проде* |
| Оплата | ЮKassa + СБП — *фаза 4* |
| SMS-OTP | SMS Aero/SMSC — *на проде; локально email-OTP* |

## Поток гостя (реализован)

```
QR/ссылка → /j/:code (лендинг, public-event)
         → /j/:code/join (имя + согласие → signInAnonymously + join-event)
         → /j/:code/camera (getUserMedia → фильтр → upload-url → PUT → confirm)
         → /j/:code/gallery (listPhotos под RLS → photo-url подписанный)
```

## Ключевые архитектурные решения

- **Хост** — телефон+OTP. **Гость** — анонимная сессия Supabase (`signInAnonymously`).
- Вступление, выдача upload-URL, согласия — через **Edge Functions (service role)**, чтобы безопасно проверять лимиты/писать журнал. CRUD-чтение — PostgREST под RLS.
- **«Проявка»**: чужие фото видны только после `events.reveal_at` — логика в RLS-политике `photos` + функция `is_event_revealed`. Дублируется в `photo-url` (ручная проверка под service-role).
- Фильтр применяется **на клиенте до загрузки**; оригинал без фильтра не хранится (минимизация данных, 152-ФЗ).
- Фото — **только подписанные URL с TTL** (бакет приватный).

## Структура репозитория

```
supabase/migrations/   — SQL (таблицы, RLS, функции)
supabase/functions/    — Edge Functions (Deno): join-event, upload-url,
                         confirm-upload, photo-url, public-event
supabase/tests/        — rls_reveal_test.sql
web/                   — Next.js PWA (гость)
docs/session-vault/    — этот конспект
```
