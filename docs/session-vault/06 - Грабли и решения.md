# 06 — Грабли и решения

[[00 - Карта проекта|← к карте]]

Самое ценное из сессии — на чём застряли и как вышли. Чтобы не наступать снова.

## 🔴 Сборка Next виснет (главная потеря времени)

**Симптом:** `next build` бесконечно висит на «Creating an optimized production build», CPU idle. То же с `next dev` и `tsc`.

**Что оказалось — два независимых фактора:**
1. **webpack-бандлер Next 15.5.19 виснет** в этом окружении (Node / macOS 26.1 arm64). → **Решение: Turbopack.** `next build --turbopack` собирает за ~17с. В `package.json` переведено на turbopack.
2. **Забитый своп.** 8ГБ RAM, при открытых Chrome + VS Code + Docker своп забивался до 90% (10ГБ), и виснул даже `tsc`. → **Решение: перезагрузка Mac** обнуляет своп.

**Диагностика, которая сработала:** изоляция — собирать по нарастающему набору файлов с таймером; `sysctl vm.swapusage` (used≈total = беда); сравнение webpack vs turbopack.

**Вторичное:** в `lib/supabase.ts` → `createClient<any,any,any>` обрезает тяжёлый вывод типов PostgREST (страховка от зависания tsc).

> [!tip] Память [[kadr-web-node-version]] (в auto-memory Claude) хранит это лечение.

## 🟡 Фото не отображались в галерее (ERR_NAME_NOT_RESOLVED)

Подписанный URL от Storage содержал внутренний хост `http://kong:8000` (Docker-сеть), браузер его не резолвит. → **Решение:** `toPublicUrl()` в `photo-url`/`public-event` подменяет origin на `PUBLIC_STORAGE_URL`. На проде no-op.

## 🟡 Камера на телефоне «не поддерживается браузером»

Мобильные браузеры блокируют `getUserMedia` без **HTTPS**. По `http://IP:3000` камера не включается. → **Решение:** локальный HTTPS через **caddy** (`tls internal`) на портах 8443 (web) / 8444 (Supabase API) + установка root-CA caddy на телефон. После этого камера на телефоне заработала. ✅

**Грабли по пути:**
- `mkcert -install` требует sudo (нет в неинтерактивном шелле) → использовали `caddy` с `local_certs`/`skip_install_trust`, CA ставится на телефон вручную.
- порт 443 привилегированный (sudo) → взяли 8443/8444.
- cloudflared-туннели (`trycloudflare`) в этой сети нестабильны — рвутся по DNS/SRV (`i/o timeout` на `[::1]:53`). `--edge-ip-version 4` помогает запуститься, но соединение всё равно отваливается. → отказались в пользу caddy+Wi-Fi.

## 🟢 Латентный баг в RLS (поймали рано)

В `0001` политика `events_guest_select` имела неквалифицированный `id` — связывался с `guests.id`, а не `events.id`, фильтр `g.event_id = g.id` всегда ложь → гость не видел своё событие. Фикс — `0002`.

## 🟢 Гонка на лимите гостей
