# 07 — Найденные баги (бэклог)

[[00 - Карта проекта|← к карте]]

Находки security-agent и qa-reviewer по фазе 3. **Не блокируют демо** (поток работает), но чинить до прода. Порядок по приоритету.

## HIGH — ✅ ЗАКРЫТО (коммит 500aeea)

- [x] **`confirm-upload` ставит `uploaded=true` без проверки файла в Storage.** → Теперь `storage.list` проверяет объект, иначе `409 not_uploaded`. Проверено в браузере.
- [x] **`photo-url` выдаёт URL для фото с `uploaded=false`.** → Теперь `404` для незагруженных. Проверено.
- [x] **Edge case §9 п.3 (`starts_at`).** → `upload-url` → `409 event_not_started`; `join-event` отдаёт `starts_at`; лендинг «Событие начнётся …»; камера блокирует съёмку. `starts_at` в `GuestSession`. Проверено в браузере (до старта upload-url не вызывается).
- [x] **(MEDIUM, заодно) `shots_left` декрементировался при провале confirm.** → Декремент только при `confirmResult.ok`; обработан код `not_uploaded`.

## MEDIUM

- [ ] **Нет серверной валидации content-type/размера файла** (§9 п.11). Можно через PUT залить не-изображение или >12МБ напрямую. → `size_limit` + `allowed_mime_types` на бакете `event-photos`.
- [ ] **`shots_left` декрементируется даже при провале `confirm-upload`.** Счётчик врёт, сообщение «появится позже» обещает несуществующее фото. Файл: `camera/page.tsx`. → Не уменьшать при `!confirmResult.ok` + ретрай.
- [ ] **«Переснять» оставляет мусорные `photos` (uploaded=false)** — DELETE не шлётся. → эндпойнт удаления или cron-очистка.
- [ ] **`reveal_at` в галерее берётся из sessionStorage** (момент join), не синхронизируется. Если хост сдвинул проявку — плашка врёт (RLS при этом корректен). → перечитывать `reveal_at`.
- [ ] **sessionStorage fallback не реализован** (комментарий обещает URL-параметры). В Private Mode → «Сессия не найдена» после вступления. Файл: `join/page.tsx`.

## LOW / на прод

- [ ] DELETE-политика на `storage.objects` для отзыва согласия + retention.
- [ ] Сузить CORS `Allow-Origin` до домена PWA.
- [ ] Убрать localhost-fallback в `toPublicUrl` на проде (требовать env).
- [ ] Очистка кэша подписанных URL при logout/смене события.
- [ ] Проверить, что Storage-политика `to authenticated` покрывает анонимных гостей в используемой версии Supabase.
- [ ] Добавить ESLint-конфиг (сейчас `next lint` не настроен).
