# 08 — Как запустить локально

[[00 - Карта проекта|← к карте]]

> [!warning] Перед сборкой web — Node 22, не системный 25
> ```
> export NVM_DIR="$HOME/.nvm"; . "$NVM_DIR/nvm.sh"; nvm use 22.22.2
> ```
> И помни: память забивается → если виснет, перезагрузи Mac ([[06 - Грабли и решения]]).

## 1. Supabase (БД + Auth + Storage + Edge Functions)

```bash
cd /Users/pavelhvostov/Desktop/photo
open -a Docker            # Docker Desktop не в автозапуске после ребута
# дождаться: docker info
supabase start            # поднимает локальный стек
supabase db reset         # применить миграции 0001-0003 (Studio: 127.0.0.1:54323)
supabase functions serve --no-verify-jwt   # Edge Functions
```

## 2. Гостевой web

```bash
cd web
npm install               # первый раз
npm run dev               # http://localhost:3000 (turbopack)
npm run build             # production-сборка (тоже turbopack)
```

## 5. Просмотр с телефона (камера → нужен HTTPS)

```bash
# IP Mac: ipconfig getifaddr en0  (был 192.168.50.124)
# caddy HTTPS-прокси (web:8443 → 3000, api:8444 → 54321):
caddy run --config .https/Caddyfile
# functions с публичным storage-хостом:
PUBLIC_STORAGE_URL="https://<IP>:8444" supabase functions serve --no-verify-jwt
# .env.local: NEXT_PUBLIC_SUPABASE_URL=https://<IP>:8444
# на телефоне: установить root-CA caddy (~/Library/Application Support/Caddy/.../root.crt),
# открыть https://<IP>:8443/j/k7p2qx
```
