## Ближайшее (закрыть хвосты фазы 3)

Из [[07 - Найденные баги (бэклог)]] — HIGH в первую очередь:

- [ ] `photo-url`: фильтр `uploaded=true`.
- [ ] `starts_at`: блокировка вступления/съёмки до старта + обратный отсчёт на лендинге.
- [ ] серверная валидация размера/типа файла (лимиты бакета).

## Фаза 4 — Биллинг + проявка (cron)

- pg_cron: авто-проявка (`reveal_at <= now()` → `status='revealed'`) каждые 5 мин.
- pg_cron: retention-метка (`expires_at <= now()` → `status='deleted'`).
- Edge Function `purge-expired` — физическая чистка Storage.
- ЮKassa: `billing/checkout` + вебхук (идемпотентность по `provider_id`), апгрейд плана события.
- Edge Functions приватности: `privacy-consent`, `deletion-request` (отзыв согласия → каскадное удаление фото + объектов Storage).

## Фаза 5 — iOS-приложение хоста (SwiftUI)

- Логин по телефону+OTP (на проде — российский SMS).
- Создание события, QR/ссылка, выбор стиля камеры/лимита/времени проявки.
- Камера хоста (Core Image фильтры), галерея с «по гостям» + избранное.
- WebView ЮKassa для оплаты.
- ⚠️ Собирать и проверять только в Xcode на Mac (из этой среды не запустить).
