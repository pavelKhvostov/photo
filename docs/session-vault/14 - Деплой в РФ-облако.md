# 14 — Деплой в РФ-облако

[[00 - Карта проекта|← к карте]]

Подготовлена инфраструктура для развёртывания (папка `deploy/`). **Реальный деплой выполняет владелец** — нужны доступы к РФ-облаку, домену, S3, которые не передаются.

## Что в `deploy/`

| Файл | Назначение |
|---|---|
| `DEPLOY.md` | главная инструкция: 11 шагов + чеклист 152-ФЗ |
| `.env.production.example` | все секреты прода (шаблон) |
| `gen-keys.mjs` | генератор ANON/SERVICE_ROLE из JWT_SECRET (проверен) |
| `Caddyfile.production` | reverse-proxy + авто-TLS + CSP (валиден) |
| `apply.sh` | миграции + функции + app_config (синтаксис ок) |
| `README.md` | указатель |

## Стек прода (всё в РФ — 152-ФЗ)

- **VM** Yandex/VK Cloud/Selectel (Ubuntu, Docker) — self-hosted Supabase.
- **Object Storage** РФ (Yandex/VK S3) — приватный бакет, шифрование at-rest.
- **Caddy** — TLS (Let's Encrypt) на kadr.ru (web) и api.kadr.ru (Supabase Kong).
- **web** Next.js (turbopack-сборка), **iOS** через Xcode/App Store.
- SMS-OTP — российский (SMS Aero/SMSC) через Auth hook.

## Порядок (кратко)

1. VM + S3 + домены в РФ-облаке.
2. self-hosted Supabase (docker-compose из репо supabase/docker).
3. `JWT_SECRET=$(openssl rand -hex 32)` → `node gen-keys.mjs` → ANON/SERVICE ключи.
4. S3-бэкенд Storage на РФ Object Storage.
5. `bash deploy/apply.sh` — миграции 0001-0007 + 11 функций + app_config.
6. web build+start, Caddy с TLS.
7. iOS: AppConfig на прод-домен, сборка в Xcode.
8. Чеклист 152-ФЗ ([[13 - Прод-готовность и env-чеклист]]).

## Обязательные env на проде

Полный список — [[13 - Прод-готовность и env-чеклист]]. Критично: `KADR_ENV=production`,
`PUBLIC_STORAGE_URL`, `ALLOWED_ORIGINS`, `PURGE_SECRET` (+ дублировать в `app_config`).

## Что НЕ входит

- Биллинг ЮKassa (план free, отложено).
- Организационное 152-ФЗ (вне кода): уведомление РКН, политика ПДн, оценка УЗ,
  меры ФСТЭК №21 — подтверждает специалист по ПДн перед запуском с реальными ПДн.
